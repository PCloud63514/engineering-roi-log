# Engineering ROI Log

이 레포지토리는 업무 개선 과정에서의 의사결정과 실행 결과를 기록하기 위한 공간이다.

단순한 결과 기록이 아니라 다음을 중심으로 정리한다:

- 어떤 문제를 발견했는가
- 왜 이 문제가 중요한가 (비즈니스/엔지니어링 관점)
- 어떤 고민과 선택지가 있었는가
- 최종적으로 어떤 결정을 했는가
- 실행 결과로 어떤 개선이 있었는가 (ROI 포함)

이를 통해 다음을 목표로 한다:

- 반복 가능한 개선 패턴 축적
- 의사결정 근거 명확화
- 성과(ROI) 기반 회고 가능
- 향후 유사 문제 해결 속도 향상

---

## 📂 구조

각 개선 사례는 하나의 디렉토리로 관리한다.

```
 [번호]_[주제]/
└── README.md
```
**예시:**
```
engineering-roi-log/
├── README.md
├── TEMPLATE.md
├── 01_배포속도개선/
│   └── README.md
├── 02_쿼리최적화/
│   └── README.md
├── 03_캐시도입/
│   └── README.md
```

## ✍️ 작성 원칙

- 결과보다 **과정(사고 흐름)** 중심으로 작성
- 가능하면 **정량적 지표(시간, 비용, 성능)** 포함
- ROI는 추정이라도 명시적으로 계산
- 실패/미완료 사례도 기록 (중요 자산)

---

## 🎯 기록 범위

- 성능 개선
- 비용 절감
- 운영 효율화
- 개발 생산성 향상
- 장애 예방/감소
